# 📖 樓層平面 AutoCAD 轉換工具 - 完整使用指南

## 🎯 系統概述

這是一個專為室內設計師打造的智能工具，可以將手繪或掃描的丈量圖自動轉換成 AutoCAD DXF 檔案。

**核心特性：**
- ✅ 自動牆面線條檢測
- ✅ 尺寸標註識別
- ✅ 門窗位置識別
- ✅ 顏色標註分類（紅色、綠色）
- ✅ DXF 檔案生成
- ✅ 友好的 Web 界面

## 📸 工作流程

### 步驟 1：準備丈量圖

你的丈量圖應該包含：

```
┌─────────────────────────────────┐
│  198.7          49.4     287.6  │
│  ┌──────────────────────────────┤  ← 頂部尺寸標註
│  │  ROOM A      冷氣套管          │
│  │  H 280.5     水管              │
│  │  ├─────────────────────────┤  │  ← 紅色：天花板高度
│  │  │                         │  │     綠色：設施位置
│  │  │                         │  │
│  │  │ ○ (門)                 │  │  ← 圓圈：門窗
│  │  │                         │  │
│  └─────────────────────────────┘
│            ↑ 邊界線（黑色）
└─────────────────────────────────┘
```

**最佳實踐：**
- 📷 使用 2000x1500px 以上分辨率
- 🖊️ 用深黑色或粗筆標註
- 📏 清晰標註所有尺寸
- 🔄 避免圖紙傾斜或透視失真
- 💡 充足的光線，避免陰影

## 💻 Web 界面使用

### 上傳丈量圖

1. **點擊上傳區域**或拖拽圖片
   - 支援 JPG、PNG 格式
   - 最大 50MB（通常 1-5MB 即可）

2. **預覽圖片**
   - 確認圖片清晰完整
   - 檢查是否傾斜

### 設定轉換參數

#### 基本設定（重要！）

**標度尺 (cm/pixel)**
- 最關鍵的參數
- 決定最終尺寸的準確性
- 計算方法：
  ```
  標度尺 = 實際尺寸(cm) ÷ 圖片像素數
  
  例子：門寬 90cm，圖上 180px
  標度尺 = 90 ÷ 180 = 0.5 cm/px
  ```

**輸出單位**
- cm（公分）- 推薦，最常用
- m（公尺）
- mm（毫米）

**牆厚 (cm)**
- 預設：15 cm
- 台灣標準：15-20 cm
- 用於繪製牆面寬度

**門寬 (cm)**
- 預設：90 cm
- 標準單扇門：80-90 cm
- 雙扇門：150-180 cm

#### 進階設定（可選）

**OCR 語言**
- 繁體中文（推薦）
- 簡體中文
- 英文

**邊界檢測靈敏度（1-200）**
- 預設：50
- 提高：檢測更多細節（可能包含雜訊）
- 降低：只檢測主要邊界

**最小線條長度 (pixel)**
- 預設：20
- 用於過濾過短的線條碎片

**自動檢測選項**
- ☑️ 自動檢測門窗（推薦開啟）
- ☑️ 自動檢測牆面（推薦開啟）

### 開始轉換

1. 點擊「**開始轉換**」
2. 系統開始處理（通常 5-30 秒）
3. 查看進度條和實時消息

### 查看識別結果

結果頁面顯示：
```
識別的尺寸標註：10 個
  • 198.7 cm
  • 49.4 cm
  • 287.6 cm
  ...

檢測的牆面：138 面
（線條檢測結果）

檢測的門窗：5 個
（圓圈聚類結果）
```

### 下載 DXF 檔案

1. 檢查結果滿意後，點擊「**下載 DXF 檔案**」
2. 檔案保存為 `floorplan.dxf`
3. 可用以下軟件打開：
   - AutoCAD
   - LibreCAD
   - Inkscape
   - CorelDRAW
   - Visio

### 下載識別報告

- JSON 格式的詳細識別數據
- 可用於進一步分析或手動編輯
- 文件名：`recognition-report.json`

## 📐 標度尺計算指南

### 方法 1：使用已知尺寸

**步驟：**
1. 在丈量圖上選擇一個清晰標註的尺寸
2. 用標尺測量圖紙上該尺寸對應的長度
3. 計算比例

**例子：**
```
已知：門寬 = 90 cm（標準尺寸）
測量圖紙：門寬在圖上 = 180 mm = 18 cm
標度尺 = 90 cm ÷ 180 mm = 0.5 cm/mm = 50 cm/cm

或用像素：
如果圖片中門寬 = 180 px
標度尺 = 90 cm ÷ 180 px = 0.5 cm/px
```

### 方法 2：多點驗證

**推薦方式（最準確）：**

1. 選擇 3-5 個不同位置的標註尺寸
2. 分別計算標度尺
3. 取平均值

```
尺寸1：門寬 90cm，圖上 180px → 0.5
尺寸2：房寬 400cm，圖上 800px → 0.5
尺寸3：樑長 200cm，圖上 400px → 0.5

平均標度尺 = 0.5 cm/px ✓ 一致性好
```

### 標度尺對照表

| 原始圖紙大小 | 常用標度尺 | 應用場景 |
|-------------|----------|--------|
| A4 白紙 | 1.0-2.0 | 小空間、精細圖 |
| A3 白紙 | 0.5-1.0 | 普通房間 |
| A2 白紙 | 0.2-0.5 | 大房間、整層樓 |
| 手機拍攝 | 0.3-0.8 | 依實際測量結果 |

## ⚠️ 常見問題和解決方案

### Q1: 標度尺怎樣設置才準確？

**A:** 使用多點驗證法：
1. 從圖上選 3 個不同的標註尺寸
2. 分別測量實際距離和圖片距離
3. 計算比例，取平均值
4. 誤差控制在 ±5% 內

### Q2: 為什麼有些尺寸沒被識別？

**A:** 可能原因和解決方案：
- 📷 圖像清晰度不足 → 重新拍攝
- 🖊️ 筆跡太淡 → 數字標註不清 → 重新標註
- 🔤 文字傾斜 → 圖片傾斜 → 調整圖片角度
- 🌫️ 背景複雜 → 遮擋文字 → 清理背景

### Q3: 牆面檢測太多或太少？

**A:**
- **檢測太多**（包含背景線條）：
  - 降低「邊界檢測靈敏度」
  - 預設 50，試試 30-40

- **檢測太少**（漏掉重要線條）：
  - 提高「邊界檢測靈敏度」
  - 預設 50，試試 60-80

### Q4: DXF 檔案怎樣在 AutoCAD 中使用？

**A:**
1. 打開 AutoCAD
2. File → Open → 選擇 `floorplan.dxf`
3. 檢查圖層和實體
4. 手動調整或添加細節
5. 另存為 DWG 格式

### Q5: 識別結果不滿意怎辦？

**A:** 嘗試以下方法：
1. **重新調整標度尺** - 最常見的原因
2. **改變圖像預處理** - 調整靈敏度
3. **重新拍攝** - 確保清晰無傾斜
4. **手動編輯** - 下載後在 CAD 中修改
5. **查看識別報告** - 了解系統的識別過程

## 📊 輸出 DXF 檔案結構

生成的 DXF 包含以下圖層：

```
LAYERS
├── WALL（牆面線條）
│   ├── 線條 1
│   ├── 線條 2
│   └── ...
├── DOOR（門窗圓圈）
│   ├── 圓圈 1
│   ├── 圓圈 2
│   └── ...
└── DIMENSIONS（尺寸標註）
    ├── 尺寸 1
    ├── 尺寸 2
    └── ...
```

**在 AutoCAD 中的用途：**
- 可分別隱藏/顯示各圖層
- 可按圖層改變顏色和線型
- 可進行進一步的修改和設計

## 🔧 技術詳情

### 處理管道

```
上傳圖片
   ↓
加載為 Jimp 圖像
   ↓
┌─ 線條檢測（牆面）
├─ 顏色分析（文字區域）
└─ 圓形聚類（門窗）
   ↓
尺寸識別（OCR/基礎識別）
   ↓
結果整合
   ↓
DXF 生成
   ↓
下載檔案
```

### 使用的庫

- **jimp** - 圖像處理
- **tesseract.js** - OCR 文字識別
- **express** - Web 伺服器
- **cors** - 跨域支援
- **multer** - 檔案上傳

### API 端點

| 端點 | 方法 | 說明 |
|------|------|------|
| `/` | GET | Web 界面 |
| `/api/process` | POST | 處理圖像 |
| `/api/generate-dxf` | POST | 生成 DXF |
| `/api/health` | GET | 健康檢查 |
