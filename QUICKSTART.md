# 🚀 快速開始指南

## 系統要求

- Node.js >= 18
- npm

## 1️⃣ 安裝和啟動

```bash
# 安裝依賴
npm install

# 啟動伺服器
npm start
```

服務器將在 `http://localhost:3000` 啟動

## 2️⃣ 使用步驟

### 步驟 1：準備丈量圖
- 用手機或掃描器拍攝/掃描你的丈量圖
- 確保圖片清晰，尺寸標註易於識別
- 推薦 JPG 或 PNG 格式

### 步驟 2：上傳圖片
1. 打開 `http://localhost:3000`
2. 拖拽圖片或點擊上傳按鈕
3. 系統會展示預覽

### 步驟 3：調整轉換參數

**基本設定（重要！）**
- **標度尺**：這是最關鍵的參數
  - 例如：如果實際寬度是 400cm，圖片寬度是 800 像素
  - 標度尺 = 400cm ÷ 800px = 0.5 cm/px
  - 測量方法：
    1. 用捲尺測量丈量圖上標註的一個已知尺寸的實際距離
    2. 用像素測量工具（或 Photoshop）測量該尺寸對應的圖片像素數
    3. 標度尺 = 實際距離(cm) ÷ 圖片像素數

- **輸出單位**：選擇 cm、m 或 mm（推薦 cm）
- **牆厚**：通常 15-20 cm
- **門寬**：標準門寬 80-90 cm

**進階設定（可選）**
- 保持默認設置通常效果最好
- 如果牆面檢測不好，調整「邊界檢測靈敏度」

### 步驟 4：開始轉換
1. 點擊「開始轉換」按鈕
2. 等待 OCR 識別完成（可能需要 10-30 秒）
3. 查看識別結果

### 步驟 5：檢查結果
系統會顯示：
- ✅ 識別的尺寸數量
- ✅ 檢測的牆面數量
- ✅ 識別的門窗位置

### 步驟 6：下載 DXF
1. 如果結果滿意，點擊「下載 DXF 檔案」
2. 檔案會以 `floorplan.dxf` 保存
3. 用 AutoCAD 或其他 CAD 軟件打開

## 📐 標度尺計算示例

假設你的丈量圖上：
- 標註的門寬是 90cm
- 測量圖片中該門的寬度是 180 像素

**計算方法**：
```
標度尺 = 90 cm ÷ 180 px = 0.5 cm/px
```

## 🎨 識別顏色說明

根據你的標註習慣，工具識別：

| 顏色 | 含義 | 例子 |
|------|------|------|
| 黑色 | 牆面和邊界 | 牆線 |

## ⚠️ 常見問題

### Q: 為什麼沒有識別到尺寸？
**A:** 
- OCR 需要清晰的文字，確保圖像質量足夠好
- 嘗試改變圖像角度（避免歪斜）
- 確保數字是清晰的黑色或深灰色

### Q: 標度尺怎樣設置最準確？
**A:**
- 多次丈量同一個已知尺寸來驗證
- 用標尺或捲尺在紙上精確測量
- 推薦測量 2-3 個不同的尺寸來取平均值

### Q: 牆面檢測不準確？
**A:**
- 調整進階設定中的「邊界檢測靈敏度」
- 增加敏感度值可以檢測更多線條
- 如果背景太複雜，可能需要更清晰的圖片

### Q: 可以手動編輯結果嗎？
**A:** 
- 目前版本支援下載 JSON 報告
- 可在 AutoCAD 中打開 DXF 檔案後手動微調
- 未來版本將支援在 Web 上手動編輯

## 💡 最佳實踐

1. **拍攝技巧**
   - 確保光線充足，避免陰影
   - 平行拍攝，避免傾斜或透視失真
   - 分辨率至少 2000x1500 像素

2. **標註清晰**
   - 使用粗黑色筆標註尺寸
   - 避免交叉塗寫
   - 清楚標註所有房間尺寸

3. **標度尺驗證**
   - 測量至少 3 個不同位置的尺寸
   - 驗證識別結果是否與實際相符
   - 必要時調整標度尺後重新轉換

## 📞 技術支持

系統日誌位置：`/tmp/server.log`

如需調試，可查看：
- 伺服器日誌了解處理過程
- 前端瀏覽器控制台（F12）了解前端錯誤
- 下載 JSON 報告查看詳細識別結果
